# Web App Architecture

## Current Structure

```
apps/web/src/lib/
├── api/                        # HTTP client (implemented)
│   ├── client.ts               # HttpClient class
│   ├── errors.ts               # HttpError, AuthError, NetworkError, RetryError
│   ├── types.ts                # HttpClientConfig, RequestOptions
│   ├── mutator.ts              # Orval integration
│   ├── generated/              # Auto-generated by Orval
│   └── index.ts
│
├── modules/                    # Domain modules (см. Module Pattern ниже)
│   ├── auth/
│   ├── groups/
│   ├── movies/
│   └── health/
│
├── compositions/               # Cross-module operations (empty)
│   └── index.ts
│
├── utils/                      # Utilities
│   ├── logger.ts
│   └── index.ts
│
└── index.ts                    # Barrel export
```

## Routes Structure

```
apps/web/src/routes/
├── (app)/                      # Protected routes (TODO)

├── +layout.svelte              # Root layout
├── +page.svelte                # Landing
└── +error.svelte               # Error page
```

## Dependency Rules

```
routes/         → modules/*, api/*
modules/*       → api/*, utils/*
api/utils/types → only each other
```

**Forbidden:**
- Modules importing other modules directly (use compositions)
- Circular dependencies

## Module Pattern

Каждый модуль организован в подпапки по ответственности:

```
modules/{name}/
├── api/                    # HTTP запросы
│   ├── {name}.api.ts       # Функции для работы с API
│   └── index.ts            # Реэкспорт
│
├── stores/                 # Реактивные сторы
│   ├── {entity}.store.svelte.ts  # Store для сущности
│   └── index.ts            # Реэкспорт
│
├── components/             # Svelte компоненты модуля
│   ├── Component.svelte    # UI компонент
│   ├── Component.types.ts  # Типы для компонента (если нужны)
│   └── index.ts            # Реэкспорт
│
├── types/                  # TypeScript типы
│   ├── {name}.types.ts     # Локальные типы (не из generated)
│   └── index.ts            # Реэкспорт
│
├── validation/             # Валидация форм
│   ├── {name}.validation.svelte.ts  # Схемы и хелперы
│   └── index.ts            # Реэкспорт
│
├── config/                 # Конфигурация (опционально)
│   ├── {name}.config.ts    # Константы и настройки
│   └── index.ts            # Реэкспорт
│
└── index.ts                # Public API модуля
```

### Назначение подпапок

| Папка | Содержимое | Примеры файлов |
|-------|------------|----------------|
| `api/` | HTTP-запросы к бэкенду | `auth.api.ts`, `groups.api.ts` |
| `stores/` | Реактивные сторы с query/mutation | `auth.store.svelte.ts`, `groups-list.store.svelte.ts` |
| `components/` | Svelte-компоненты модуля | `GroupForm.svelte`, `MovieCard.svelte` |
| `types/` | TypeScript типы и интерфейсы | `auth.types.ts`, `movies.types.ts` |
| `validation/` | Zod-схемы, валидаторы форм | `auth.validation.svelte.ts` |
| `config/` | Константы, настройки модуля | `health.config.ts` |

### Пример полной структуры модуля

```
modules/groups/
├── api/
│   ├── groups.api.ts       # getGroups, getGroup, createGroup, updateGroup
│   └── index.ts
├── stores/
│   ├── groups-list.store.svelte.ts   # Список групп (только чтение)
│   ├── group-item.store.svelte.ts    # Одна группа + mutations
│   └── index.ts
├── components/
│   ├── GroupForm.svelte    # Форма создания/редактирования
│   ├── GroupForm.types.ts  # Props, режимы формы
│   └── index.ts
├── validation/
│   ├── groups.validation.svelte.ts   # Zod-схемы, мапперы
│   └── index.ts
└── index.ts                # Экспорт stores, components, types
```

### Правила импортов

```typescript
// ❌ ИЗВНЕ модуля — запрещено импортировать из внутренних файлов
import { groupStore } from '$lib/modules/groups/stores/group-item.store.svelte';

// ✅ ИЗВНЕ модуля — только через публичный index
import { groupStore, groupsStore, GroupForm } from '$lib/modules/groups';

// ✅ ВНУТРИ модуля — прямые импорты из подпапок
// В stores/group-item.store.svelte.ts:
import { getGroup, createGroup } from '../api/groups.api';
import type { GroupFormData } from '../validation/groups.validation.svelte';
```

**Подробнее:** [Query Store Pattern](query-store-pattern.md) — паттерн использования `createQuery` и `createMutation` в сторах.

## Component Layers

| Layer | Location | Purpose |
| ----- | -------- | ------- |
| API Client | `lib/api/client.ts` | HTTP with auto-refresh, retries |
| Generated | `lib/api/generated/` | Orval-generated API functions and DTOs |
| Modules | `lib/modules/*/` | Domain logic (auth, movies, groups) |
| Compositions | `lib/compositions/` | Cross-module operations |
| Utils | `lib/utils/` | Logger, formatters |

## Generated Types (from Orval)

**Auth:** `AuthLoginDto`, `AuthRegisterDto`, `AuthResponseDto`, `CsrfResponseDto`

**Users:** `UserCreateDto`, `UserResponseDto`, `UserUpdateDto`

**Movies:** `MovieCreateDto`, `MovieResponseDto`, `MovieUpdateDto`, `ProviderMovieSummary`, `ProviderSearchResult`, `AddMovieDto`

**Groups:** `GroupCreateDto`, `GroupResponseDto`, `GroupUpdateDto`, `GroupMemberAddDto`, `GroupMemberResponseDto`, `GroupMemberRoleUpdateDto`, `GroupMovieResponseDto`, `EditGroupMovieDto`, `TransferOwnershipDto`

**Custom Movies:** `CreateCustomMovieDto`, `CustomMovieResponseDto`, `UpdateCustomMovieDto`

## API Client Features (Implemented)

- Auto-refresh on 401
- Retry on network errors (3 attempts, 1s delay)
- CSRF token management
- Orval integration for type-safe API calls
- Timeout handling (30s default)
